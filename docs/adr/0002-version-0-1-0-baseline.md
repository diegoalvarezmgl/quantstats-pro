# Version 0.1.0 as fork baseline

QuantStats Pro starts its own release line at `0.1.0`, not continuing upstream's `0.0.x`. This signals a distinct maintained fork while staying in `0.x` semver to allow API value corrections before a `1.0.0` portfolio-ready release.

Policy: `0.1.x` = formula bugfixes and tests; `0.2.0` = reports/plots redesign; `1.0.0` = portfolio-ready milestone.

**Considered:** Continue at `0.0.82` (too easily confused with upstream maintainer) or jump to `1.0.0` (premature given pending bugfixes and visual redesign).
